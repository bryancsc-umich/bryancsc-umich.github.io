Personal Projects

Scheduler.io

Allows whitelisted members to upload simple python scripts, schedule when to run them and receive email notifications with an easy-to-use user interface.

The best part about the scheduler? It isn't tied to any platform/app/device and users don't have to go through the learning curve of learning how to set up their own service from scratch.

Built with Flask, HTML, CSS, JavaScript.

Order App

An Android app built to help restaurant owners during Covid-19. Register customer orders using the mobile app and simply click a button to automate a phone call or send an SMS to inform the customer when an order is ready.

Built with Flask and Android.

ReMi: Conversation Cooking Assistant

Anyone who cooks will realize how useful it is to have another individual helping you out. Sometimes you forget the recipe steps, sometimes you need alternative ingredients, or perhaps you need to convert between units. This project can be thought of as a specialized version of Alexa/OK Google with respect to culinary preparation. It provides a chat interface to common kitchen functionalities such as recipe walkthroughs, ingredient substitutions, unit conversions and even timers!

Built with Flask, HTML, CSS and JavaScript.

Handshake

Contracts are usually a lengthy annoying part of life. We usually communicate contracts using lengthy email threads and it's annoying to sign them. Plus, many of us would draft simple contracts during our everyday lives (such as sublease agreements and freelance work) but many of us aren't experienced in drafting contracts. Introducing Handshake (making contracts as easy as a handshake). This web-app enables people to easily draft and sign contracts and includes a feature that allow users to create and share contract templates with the rest of the user community for common scenarios.

Built with Flask and React.

Form Bot Dialogue Pipeline

Play around with the existing open source chatbot creation platforms and you will realize that many platforms place their focus on identifying a user intents. However, they aren't exactly well-designed when it comes to defining a "user path". However, a user path is usually what we need when it comes to structured data collection. Sometimes what we need is a blend between old-fashion logic and neural-network based entity extraction. This dialogue management system does exactly that. It allows you to define a specific user path the old-fashioned way while also exposing intent and entity extraction for your use.

Built with Flask and Rasa Open Source.

Natural Language Diagnosis Software

An advanced extension on the image recognition diagnosis app mentioned above. A user can input a description of a disease in natural text and the system will provide a list of top matching diseases. The underlying concept that drives this system is a boosting algorithm and the data used to train the system was collected using a web crawler. The system also has a novel query expansion mechanism that attempts to produce different re-expressions of a query such that the query does not necessarily need to have words that match a relevant disease’s description in our database.

Built with Python.
